## Archive Policy

### When to Archive

**Projects**: Archive when:
- Project is completed and delivered
- All payments received
- No ongoing maintenance expected
- 30 days have passed since completion

**Areas**: Convert to archive when:
- No longer an active responsibility
- Has been inactive for 6+ months
- Replaced by new area or process

**Resources**: Archive when:
- Technology is deprecated
- Information is outdated
- Better resources are available
- Not accessed in 12+ months

### What to Keep

- Final deliverables
- Contracts and agreements
- Financial records (7 years)
- Lessons learned documents
- Key communications
- Success metrics

### What to Delete

- Draft versions
- Redundant files
- Personal notes (if not valuable)
- Temporary files
- Low-value communications

## Retention Schedule

### Keep Forever
- Contracts and legal documents
- Tax returns and financial records
- Major project deliverables
- Lessons learned summaries

### Keep 7 Years
- Invoices and receipts
- Client communications
- Project documentation

### Keep 3 Years
- Meeting notes
- Status reports
- Internal planning docs

### Keep 1 Year
- Draft documents
- Routine communications
- Temporary files

---

## Archive Maintenance

### Annual Review
Once per year:
- Review all archived items
- Delete items past retention period
- Compress and back up remaining archives
- Update archive index

### Backup Strategy
- Local backup: External hard drive
- Cloud backup: Encrypted cloud storage
- Version control: Git for code and docs
