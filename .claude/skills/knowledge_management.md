# Knowledge Management Skill

## Description
Organizes, stores, and retrieves information, documentation, and learnings. Implements the PARA method (Projects, Areas, Resources, Archives) for personal knowledge management.

## Capabilities
- Organize information using PARA method
- Create and maintain documentation
- Build personal knowledge base
- Tag and categorize content
- Search and retrieve information
- Link related notes and documents
- Track learnings and insights
- Maintain project documentation
- Archive completed work

## Example Usage

### Add to Knowledge Base
```
Add a new resource about React performance optimization. Include key concepts, best practices, code examples, and links to documentation.
```

### Find Information
```
Find all my notes about Stripe integration, including API examples and troubleshooting tips.
```

### Document Project Learning
```
Document key learnings from the TechCorp website project, including what went well, challenges faced, and solutions found.
```

### Create Template
```
Create a project kickoff checklist template that I can reuse for future client projects.
```

## PARA Method Overview

### Projects
Active work with defined outcomes and deadlines
```
/Projects
  /Website_Redesign_TechCorp
  /Mobile_App_StartupXYZ
  /Brand_Identity_AcmeCorp
```

### Areas
Ongoing responsibilities and standards
```
/Areas
  /Client_Management
  /Business_Development
  /Professional_Development
  /Health_and_Wellness
```

### Resources
Topics of ongoing interest and reference material
```
/Resources
  /Web_Development
    /React
    /Node.js
    /CSS_Frameworks
  /Design
    /UI_UX_Principles
    /Color_Theory
    /Typography
  /Business
    /Marketing
    /Sales
    /Legal
```

### Archives
Completed or inactive items from other categories
```
/Archives
  /2024
    /Completed_Projects
    /Old_Clients
    /Outdated_Resources
```

## Best Practices

### Capture Immediately
Write things down when you learn them, don't wait

### Use Consistent Structure
Follow templates and formats for similar content

### Link Generously
Connect related notes with bidirectional links

### Review Regularly
Revisit and update notes to keep them current

### Archive Ruthlessly
Move inactive content to Archives

### Tag Thoughtfully
Use tags to create multiple access paths

### Make It Searchable
Use clear titles and descriptive content

### Keep It Simple
Don't over-organize or create too many categories

### Focus on Utility
Prioritize information you'll actually use

## Common Pitfalls

### Over-organizing
Spending more time organizing than creating

### Under-linking
Missing connections between related information

### Neglecting Archives
Keeping everything active creates clutter

### Inconsistent Naming
Makes searching and sorting difficult

### No Maintenance
Knowledge base becomes outdated and unreliable

### Hoarding Information
Keeping everything instead of what's useful

### Not Using Templates
Reinventing the wheel for similar content
